# Security Policy for J.A.R.G.I.I.N.

## Supported Versions

This repository is currently in the `1.x` line. Security fixes are expected to land on the latest `main` branch state and any active `1.x` releases.

| Version | Supported |
| ------- | --------- |
| 1.x     | :white_check_mark: |
| < 1.0   | :x: |

## Reporting a Vulnerability

Please do **not** open a public issue for suspected security vulnerabilities.

Use GitHub's private vulnerability reporting for this repository if it is available. If private reporting is not available, contact the maintainers through a non-public channel first and share only the minimum detail needed to reproduce and triage the issue.

When reporting, include:

1. A clear description of the issue and affected area.
2. Reproduction steps or a proof of concept.
3. Impact assessment, including whether secrets, transcripts, prompts, or API keys may be exposed.
4. Any suggested mitigations or workarounds already identified.

What to expect:

1. Initial acknowledgment as soon as the report is reviewed.
2. Triage to confirm severity, scope, and reproducibility.
3. Coordination on remediation and disclosure timing when the report is valid.

Please avoid including real credentials, API keys, or personal data in reports. If sensitive material is necessary for reproduction, redact it wherever possible.
